# Repository Roles

## `HelabHLC/arbe-lambda`

Role: **Scientific Foundation**

Contains normative ARBE λ*_V2 specification, deterministic reference implementation, reproducibility material, citation guidance and scientific release records.

## `HelabHLC/designfit-studio`

Role: **ARBE Core implementation and applications**

Contains Reference Engine, Atlas Repository, AtlasFit contracts, spectral primitives, controlled runtime adapters, public documentation and ARBE DesignFit Studio.

## Authority rule

Application code may implement the scientific reference but must not silently redefine it. Dataset and runtime identities must be based on manifests and checksums, not repository branches, domains or product labels.
