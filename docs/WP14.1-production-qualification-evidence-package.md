# WP14.1 — Deterministic Production Qualification Evidence Package and Verifier

## Purpose

WP14.1 turns the WP13 production qualification report into a portable, deterministic evidence package. The package preserves the normalized feasibility qualification, the embedded WP13 report, an exact artifact manifest, and cryptographic linkage between all layers.

The feature does not change feasibility evaluation, colour identity, recipe selection, manufacturing tolerances, or production-release policy.

## Contract

The package schema is `ARBE_PRODUCTION_QUALIFICATION_EVIDENCE_PACKAGE_V1` and contains:

- the normalized qualification snapshot;
- the complete `ARBE_PRODUCTION_QUALIFICATION_REPORT_V1` report;
- qualification and report fingerprints;
- one SHA-256 manifest entry for the snapshot, report, every evidence record, and every criterion record;
- an explicit claim boundary;
- a canonical package SHA-256 over the complete payload.

All arrays whose order is not semantically meaningful are normalized before hashing. The returned package is deeply frozen.

## Builder

Use `createProductionQualificationEvidencePackage(qualification)` from:

`src/core/feasibility/production-qualification-evidence-package.ts`

The builder accepts only an existing normalized `FeasibilityQualification`. It delegates report production to the WP13 builder and does not reimplement qualification policy.

## Independent verifier

Use `verifyProductionQualificationEvidencePackage(value)` from:

`src/core/feasibility/production-qualification-evidence-verifier.ts`

The verifier accepts `unknown`, performs its own envelope inspection and canonical hashing, and does not call the package builder. It evaluates:

1. schema presence;
2. package integrity;
3. embedded report integrity;
4. qualification/report linkage;
5. exact artifact-manifest coverage;
6. criterion-to-evidence references;
7. mandatory claim boundaries.

The result schema is `ARBE_PRODUCTION_QUALIFICATION_EVIDENCE_VERIFICATION_V1` with status `VERIFIED` or `REJECTED` and deterministic findings.

## Qualification coverage

The automated suite covers:

- positive package construction and verification;
- malformed input rejection;
- embedded-report tampering;
- package-hash tampering detection;
- artifact-manifest tampering;
- reproducibility across semantically equivalent input order;
- regression linkage to the WP13 report schema, fingerprint, hash, and claim boundary.

## Boundary

A verified package proves deterministic integrity and internal linkage only. It does not certify production output, approve a recipe, replace a controlled production trial, redefine colour identity, or grant production release.
