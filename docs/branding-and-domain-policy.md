# ARBE branding and domain policy

## Current rule

`ARBE` is the platform and project name. No public interface, metadata, architecture document or runtime identifier may assume ownership of a specific internet domain until that domain is confirmed and controlled by the project.

## Product and repository names

- Platform: `ARBE`
- Product: `ARBE DesignFit Studio`
- Scientific reference repository: `arbe-lambda`
- Application repository: `designfit-studio`

## Domain boundary

Domain names are deployment configuration, not scientific identity and not dataset identity.

The application must not:

- use a provisional domain as the brand name;
- encode a domain in reference IDs, dataset IDs or audit records;
- treat a branch name or hostname as a versioned data source;
- publish links to an uncontrolled domain.

## Future domain adoption

A future domain may be added only after availability, ownership and intended use have been confirmed. The domain should then be supplied through deployment configuration and canonical metadata rather than hard-coded throughout the domain core.

## Migration note

The earlier `ARBE.org` wording was a provisional project label. It has been replaced by domain-neutral `ARBE` wording because `arbe.org` is not controlled by this project.
